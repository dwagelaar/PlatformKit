Setting up a Model-Driven Software Product Line

  • Back to the PlatformKit main page

Introduction

Since there are too many ways to describe here for setting up an SPL, we will follow the Instant Messenger case study as an example. Installation instructions for this case study can be found here. Since the instant messenger is a client-side applet, we can reuse the existing model transformations for Java applets.

Core assets

The core assets for the Instant Messenger product line are stored in an Eclipse ATL project. These core assets include at least the UML model of the common architecture for all instant messenger products as well as the UML models of each optional feature. Any third-party components that must be built from source can be stored in a separate Eclipse Java project.

Products

Each instant messenger product is stored in a separate Eclipse Java project. That way, project-specific settings for compilation can be used.

We can now select and apply model transformations to the common UML model and the selected optional UML models using a build.xml Ant script. The resulting generated code can be stored in the Eclipse Java project for the relevant instant messenger product.

Later on we will create a configuration language for the instant messenger product line, which allows us to automatically generate a product-specific build.xml script with a model transformation. This step is not necessary for the purpose of PlatformKit, however.

Now that the source code for the instant messenger product is available, any additional resources can be added and packaging scripts can be written. Any reusable packaging scripts can be refactored into the main product line Eclipse project. Templates can be added to the main product line Eclipse project for product-specific building and packaging scripts.
