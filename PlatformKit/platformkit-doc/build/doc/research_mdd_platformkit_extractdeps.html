Extracting Platform Dependencies of Third-party Components

  • Back to the PlatformKit main page

Introduction

Finding out the platform dependencies for your own software models and/or source code is relatively straightforward, compared to finding out the platform depencies of third-party binaries. PlatformKit can help you to extract the platform dependencies for third-party Java jar files. It uses the UML models generated by the Jar2UML tool for this.

System Requirements

Extracting a UML model of the jar file(s)

First, we need a UML model that includes all the API dependencies of the third-party component’s jar file(s):

  • Select “File → Import...”.
  • Select “Jar2UML Import → Import Jar File Dependencies to UML Models” and click “Next” (see Fig. 1).
  • Select the component’s jar file(s) by clicking “Browse...”. Fig. 2 shows how to import the “MicroJabberWookie.jar” file, which can be found in “uml2cs-microjabberwookie/jar/” if you installed the instant messenger case study.
  • Select a target folder for your UML model.
  • Click “Finish”.

Comparing the UML model against known platforms

Now that the UML model has been stripped down to only Java API dependencies, PlatformKit can compare it against built-in UML models of various Java APIs:

  • Right-click the “.uml” file and select “PlatformKit → Determine compatibility with X...”, where X is the Java API you want to compare against (see Fig. 5).
  • An ATL query is run against the models and checks whether the selected Java API is compatible with the API dependencies we’ve extracted.
  • The result – compatible or not – is displayed in a dialog window, while the detailed comparison results can be viewed in the ATL console (see Fig. 6).

By finding out which Java APIs are compatible, you can derive which versions of particular Java API packages to use when Modelling Platform Dependencies. “MicroJabberWookie”, for example, requires “java.lang”, “java.io” and “java.util” and turned out to be compatible with all Java APIs.
